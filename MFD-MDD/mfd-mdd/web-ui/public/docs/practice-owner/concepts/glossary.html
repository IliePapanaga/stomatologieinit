Glossary

Compute

Computational resources of a provider (i.e. available memory and processor).

Environment

An environment is a logical collection of application and/or service templates. Environment lets you group a number of tested templates in one place so that you could quickly deploy or upgrade all of them without having to edit and deploy dozens of separate templates one by one.

Media format

A media format is used exclusively for information-mapping and visualization purposes (so that you could see the list of available formats at a glance): it does not affect the functionality.

Media store

A media store is a place where certain files (e.g. executable installers, .zip archives, scripts, and so on) are located. Note that P3C supports only SMB media stores, i.e. any Windows file server can be used as a media store. Media stores are visible for any host that P3C deploys.

Site

In P3C, a site is a set of computational and storage resources needed to run one or more spaces. A site is linked to a single cloud; however, you can use one cloud to run and manage multiple sites (if a site is small enough to be run by that cloud).

Space

Space is a virtual container for the software managed within a cloud. Note that each space can be linked to only one organization that runs the VMs and apps in that space.

Storage

Storage is a combination of both virtual and physical resources of a provider, e.g. VMware®, etc. that can be allocated to a site (MBytes).
