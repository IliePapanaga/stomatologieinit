Adding a service or app maintenance plan

You can add a service or application maintenance plan. You can set up various maintenance plans in order to quickly upgrade a number of application or service templates.

  1. Click Design Studio.
  2. Click the Service Maintenance Plans or Application Maintenance Plans tab.
  3. Click Add Maintenance Plan.
  4. Expand the General folder, and click Properties; specify the following:
    • Type - select the only available type, Upgrade.
    • Source - select an outdated template that you must upgrade.
    • Target - select a template whose settings will be used to update those of an outdated one.
      Tip: You can modify automatically generated data displayed in the Name field.
    • Vendor - type in a vendor's name, ID or description.
  5. Click Orchestration > Execution > Execute, and enter the script's name into the Name field, e.g. space_formatting.ps1 (note that the file-name extension is .sh for Linux- and .ps1 for Windows-compatible VMs).
    Tip: If you want to automatically restart the VM after a script was executed, select the Required Reboot check-box; remember to specify execution and post-execution scripts when needed.
  6. Click OK.
    Note: If you want to create another version of a maintenance plan, select it in the Maintenance Plan pane, and click the Save as New Plan button on the toolbar.
You can now select a maintenance plan, click Test Plan on the toolbar, and then select the environment and service to be tested. If the test is successfully passed, "Success" will be displayed in the Status column next to the corresponding maintenance plan (Lab > Activity History).
