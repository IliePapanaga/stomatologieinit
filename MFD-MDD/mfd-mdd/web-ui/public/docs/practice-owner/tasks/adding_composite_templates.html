3.2.1 Add/Edit Permanent Posting

A composite type of a service or application template enables you to add multiple topology elements (that is, templates), and establish dependencies between those templates from one workspace (Service Management > Topology). That is the only difference between a plain and a composite template type. You might need to create a composite template if, for example, you have multiple VMs running, say, MySQL (dependent ones) and a mainframe VM where Tomcat is installed,and you need the dependent VMs to automatically inherit and activate the settings of the mainframe one. You could then establish a dependency between the mainframe VM (endpoint), and dependent VMs (by clicking Add Dependency). Then the endpoint VM's configuration would be automatically linked to, triggered, and 'pushed down' to dependent VMs when the template's corresponding service gets provisioned.

Tip: When you are setting up a service template, especially a composite one, you need to set up a deployment group the template belongs to. A composite service template can include multiple deployment profiles which, in turn, can contain a number of deployment groups intended for a particular purpose. A deployment group is a collection of the same type of VMs to be provisioned. Note that the deployment group's settings take precedence over and supersede the settings of an individual service's OS.
  1. Click Design Studio.
  2. Click either the Service Templates or Application Templates tab, select the plain template the composite one will be based on, and click Test Template.
  3. When testing is completed, select the template, and click Publish Template.
  4. When published (the template's state is changed to "Active"), click Add Template > Add Composite Template.
  5. Expand the Service Management folder, and click Topology.
    The published template(s) will be displayed in the Topology Components pane.
  6. From the Topology Components pane, expand the folder(s), then drag and drop the template on to the topology workspace.
    Tip: The Topology "drawing board" is a visual aid that showcases configuration dependencies and connections between various templates and their services. Note that, to reduce visual clutter, you can re-size the templates being displayed.
  7. Click Add Endpoint, and type in the VM's name.
  8. From the Endpoint Type list, choose the type that you pre-configured.
    Note: If you want to create a 'tunnel' through the network's gateway for this endpoint to be accessible to all users outside the firewalled, isolated network, select the User Accessible check box.
  9. From the Topology Components pane, drag and drop another template.
  10. On the toolbar, click Add Dependency, type in its descriptive name, and select its endpoint type (to PLACEHOLDER, select the Optional check-box).
    Warning: The endpoint types of an endpoint and a dependency that you are adding must match.
  11. Point to the endpoint's connector circle, and wait until a green circle is displayed around it: click the green circle, and then drag its dotted line to the dependency template as illustrated in this tutorial screen-cast (in your web-browser, click Open Link in New Tab).
    Fastpath: When using a composite template, you can enable the toolbar button that lets you quickly add a media file: make sure you have selected Process Customizations from the Operation Type list and added this type. You can then expand the Service Management folder, click Topology, and use the Manage Customize button on the toolbar to quickly upload a media file to a service or application template.
  12. In the Summary pane, click OK.
The dependent template (and its services) is now linked to the endpoint template (you can link multiple dependent templates to 1 endpoint): the dependent templates' services or apps will be started as soon as the endpoint template is activated, i.e. provisioned.
