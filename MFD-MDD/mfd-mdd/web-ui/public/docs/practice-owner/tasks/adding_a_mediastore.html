Adding a media store

You can set up a media store which you can use later when adding media files to a service template. A media store is, for example, a local folder whose contents is uploaded to a certain resource. A media store can contain .rar, .exe, etc. file types.

For more details, see Media store.

  1. Click Infrastructure Operations.
  2. Click the Deployment view, then the Media Stores tab, and Add Media Store.
  3. Enter the following:
    • Name - an arbitrary name.
    • Media Provider - a media provider's class, for example (P3C uses media stores via plugged-in media store providers):
      Note: You can select only one provider within a class.
      • smb - the Samba protocol; select this check-box if you want a Windows client PC to establish connection with a resource shared on a Linux network.
    • Key - an automatically generated ID.
    • Store Identification - the path to a specific folder, e.g. if you are using Samba. To an SMB provider, it is a URI to a shared folder on a Windows file server, e.g. smb://fqdn_of_fileserver/Share/P3C .
    • User name and password credentials - enter the corresponding credentials that will be used when connecting to a resource through Samba, etc., including a domain where applicable, for example: mydomain\mycomp.
  4. Click OK.
The media store you have added is available from: Design Studio > Service Templates > Add Template > Media Files > Add Media File > Register Media File > Media store.
