Restarting or abandoning an operation

You can restart an operation with the Failed, etc. status or you can abandon, i.e. cancel that operation altogether. If you click Abandon Operation, the corresponding type of an activity, for example, provisioning of a site, will be cancel both in P3C and the respective cloud provider's solution, e.g. vSphere. At times, abandoning the operation might fail: you can then select that activity and click Force Abandon Operation ̶ the operation will be cancelled and purged from P3C only, but not from a cloud provider's solution.

  1. Click Infrastructure Operations.
  2. Click the Activity view, select a failed operation, and then click Restart Operation:
    Remember: An operation get restarted based on its status, i.e. if it is "Provisioning", then it is re-provisioned; if the Status column contains "Abandoning", then it is re-abandoned, etc.
    • Simple - restarts it.
    • Custom - select the check-box next to a specific step, or steps, that you want to skip, and click OK.
Note: If for some reason a space-provisioning, etc. operation can be neither abandoned nor force-abandoned, then you can click Force Delete Operation - the corresponding space is removed from Infrastructure > Spaces.
