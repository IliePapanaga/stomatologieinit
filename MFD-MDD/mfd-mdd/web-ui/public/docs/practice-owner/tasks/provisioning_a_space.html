Registering and provisioning a space

If spaces have already been added by a system administrator, you can register a space, i.e. make it available for use with one of the Sites (to do that, right-click a space and click Register space). You can then provision a space, i.e. deploy it within the environment of a cloud provider (VMware, etc).

  1. Click Infrastructure Operations, then the Spaces tab, and right-click an available space.
    Tip: The State column contains one of the 4 possible states of a space:
    • "Configuration" - the space's set-up process is incomplete, for example, a network has not been assigned to a space, etc.
    • "Ready for provisioning" - the configuration process is complete, you can select a space and click Provision Space.
    • "Processing" - the space is being provisioned, i.e. deployed on the provider's environment.
    • "Active" - the space is up and running.
  2. Select a space with the "Ready for provisioning" state, and click Provision Space on the toolbar.
    Note: To check the progress of the space provisioning, click the Activity view: it is displayed in the Status column (for more information, click View Details).
    The state changes to "Processing", and then to "Active".
Remember: To make a space unavailable, select a space, then click Remove Space on the toolbar, and click either Delete Space (if it has been provisioned and is active) or Unregister Space: deleting a space deletes it from both a cloud solution (e.g. vSphere) and P3C, while unregistering a space removes it only from P3C.
